# Webhook Contracts

This repository forwards validated requests to external n8n workflows.

## Upload Flow (Vercel-safe)

1. Client requests a signed upload URL from `POST /api/uploads/receipt`.
2. Client uploads binary directly to Supabase Storage using the returned token.
3. Client submits expense intake to `POST /api/expenses` with receipt metadata only.

`POST /api/expenses` does not accept binary files.

## OCR Webhook

- Env var: `N8N_OCR_WEBHOOK_URL`
- Purpose: Extract and normalize receipt data from uploaded image references.
- Input schema: `ocrWebhookInputSchema` in `lib/n8n/contracts.ts`
- Output schema: `ocrWebhookOutputSchema` in `lib/n8n/contracts.ts`

## Policy Webhook

- Env var: `N8N_POLICY_WEBHOOK_URL`
- Purpose: Apply expense policies and return final decision.
- Input schema: `policyWebhookInputSchema` in `lib/n8n/contracts.ts`
- Output schema: `policyWebhookOutputSchema` in `lib/n8n/contracts.ts`

## Director Chat Webhook

- Env var: `N8N_DIRECTOR_CHAT_WEBHOOK_URL`
- Purpose: Handle director assistant analytics queries.
- Input/output schema: to be added in `lib/schemas/director.ts` and `lib/n8n/contracts.ts`.

## Shared Headers

- `Authorization: Bearer <N8N_WEBHOOK_BEARER_TOKEN>`
- `x-correlation-id` (optional on input; generated by API if absent)
- `x-idempotency-key` (optional on input; generated by API if absent)
